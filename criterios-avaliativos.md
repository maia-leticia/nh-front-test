# Critérios Avaliativos para o Projeto de Teste

- **Conversão para TailwindCSS:**
  - Todo o CSS deve ser convertido para classes utilitárias do TailwindCSS.
  - Remover arquivos CSS desnecessários após a conversão.

- **Ajuste de Semântica HTML:**
  - Utilizar tags semânticas apropriadas (header, main, footer, section, article, nav, etc).
  - Garantir acessibilidade básica (alt em imagens, labels em formulários, etc).

- **Correção de Placeholders e Conteúdo:**
  - Substituir textos genéricos por conteúdo realista ou instruções claras.
  - Ajustar imagens e ícones conforme necessário.

- **Ajuste de Espaçamentos e Layout:**
  - Corrigir margens, paddings e alinhamentos para garantir boa apresentação visual.
  - Garantir responsividade em diferentes tamanhos de tela.

- **Formatação do Código:**
  - Seguir boas práticas de indentação e espaçamento.
  - Remover código morto ou não utilizado.

- **Organização de Arquivos:**
  - Separar componentes em pastas adequadas.
  - Manter uma estrutura clara e lógica para o projeto.

- **Separação de Responsabilidades:**
  - Garantir que cada componente tenha uma responsabilidade única.
  - Evitar componentes muito grandes ou com múltiplas funções.

- **Outros Itens a Serem Corrigidos:**
  - Corrigir eventuais bugs de funcionalidade ou navegação.
  - Garantir que o projeto rode sem erros ou warnings no console.
